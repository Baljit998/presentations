EXPLOR

  • "EXplicitly defined Patterns, Local Operations, and Randomness"
  • also created by Ken Knowlton at Bell Labs in 1970s
  • probabilistic language

[source: EXPLOR ‚Äî A Generator of Images from EXplicit Patterns, Local Operations & Randomness, Proc. 9th Annual UAIDE Mtg, Ken Knowlton, 1970]

Syntax

name opcode (n,p)list-of-arguments, goto

Grid 340x240

Probabilistic language

(n,p) probabilistic indictator
(1,1) always do this operation when control gets here
(1,16) do this one with probability 1/16
(4,1) do this one every fourth time
(8,2) every eighth time through, flip a coin to decide
(X,50,l) except for the 50th, 100th, etc. times, do it

Transliteration

XL (n,p)q(xlit)goto

e.g.

XL (2,1)3(AB,CD)

Transliteration

XL for each pixel
(n,p) (2,1) every other time this line is reached
q 3 about one third
xlit (AB,CD) of all pixels with valueAs update to Bs and Cs update to Ds.

Adjacency-conditioned transliteration

AXL (n,p)nums,dirs,chst,q(xlit)goto

e.g.

AXL (1,4)234,NESW,ABCD,5(56,78)

Adjacency-conditioned transliteration

AXL for each pixel
(n,p) (1,4) if this line is run with probability of 1/4
nums 234 and if there are exactly 2, 3 or 4 neighbours of a pixel
dirs NESW in the NORTH, EAST, SOUTH or WEST positions
chst ABCD whose value is A, B, C or D
q(xlit) 5(56,78) with the probability of 1/5, update current pixel 5s to 6s or 7s to 8s
